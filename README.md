# 🛠 Сборщик выписки (XML Agregator)
[Ссылка на онлайн-версию сборщика](https://rintaru123.github.io/EBP-account-statement-packager)

**Программа разработана при поддержке Финансового отдела администрации Аликовского муниципального округа Чувашской Республики в целях автоматизации бухгалтерии в рамках проекта "Эффективный регион"**

Инструмент для автоматического объединения банковской выписки, платежных поручений (ED101/ED108) и распоряжений (РСКП) в единый документ для импорта в бухгалтерские системы (например, 1С или АЦК).

![Сборщик выписки](screenshots/1.png))

Работает прямо в браузере. Данные не отправляются на сервер.

### 🌟 Что нового в v1.37
*   **Авто-сортировка файлов:** Теперь не нужно выбирать файлы по отдельности. Просто перетащите всю папку или выделите все файлы XML сразу — программа сама определит, где выписка, а где платежки.
*   **Строгий фильтр КБК:** Улучшена логика обработки доходов. Если в назначении платежа (возмещение и т.д.) найден расходный КБК (например, оканчивающийся на 244), он игнорируется, чтобы избежать ошибок при импорте.
*   **Умное именование:** Итоговый файл сохраняется с именем `Приложение к выписке от [ДАТА].xml` для удобства архивации.
*   **Отображение даты:** Дата выписки теперь выводится в консоль лога.
*   **Умный анализ КБК** Проводится анализ КБК из назначения платежа и встраивается в принимаемую выписку

### 🚀 Как пользоваться
1. Скачайте файл `index.html`.
2. Откройте его в любом современном браузере (Chrome, Edge, Firefox).
3. Нажмите на область загрузки или перетащите в неё все XML файлы, полученные из казначейства/банка (файл выписки + файлы платежек).
4. Нажмите кнопку **"Сформировать и Скачать"**.

### 🔧 Техническая информация
*   **Платформа:** HTML5 + Vanilla JS (ES6).
*   **Зависимости:** Нет.
*   **Принцип работы:** Парсинг XML через DOMParser, сопоставление платежей по GUID/Сумме/Номеру, генерация агрегированного XML по схеме `Inf_Pay_Doc (formulars.xsd)`.
